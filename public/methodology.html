How WP-Rank Works

WP-Rank analyzes WordPress sites to understand the relationship between plugin usage and performance. Our methodology combines automated discovery, performance testing, and plugin detection to provide insights into WordPress site optimization.

Key Insight

We measure "efficiency" as the ratio of performance to plugin count, helping identify sites that achieve high performance despite using many plugins, or conversely, sites with poor performance despite minimal plugin usage.

Site Discovery

Manual Submissions

Users can submit WordPress sites for analysis through our submission form. We validate that submitted domains are accessible and appear to be WordPress sites.

Automated Discovery

Our system periodically discovers new WordPress sites through:

  • WordPress.com directory and feeds
  • WP.org showcase and community sites
  • Technical blogs and WordPress-focused websites
  • Following links from already-analyzed sites (when they link to other WP sites)

Quality Filters

We exclude staging sites, development environments, and sites that appear to be under construction or abandoned.

Performance Analysis

Google PageSpeed Insights

We use Google's PageSpeed Insights API to measure real-world performance metrics:

  • Performance Score: Overall score from 0-100
  • Largest Contentful Paint (LCP): Time to render the largest element
  • Cumulative Layout Shift (CLS): Visual stability measurement
  • Total Blocking Time (TBT): Time the main thread is blocked

Mobile Focus

All performance tests are conducted using mobile device simulation, reflecting the mobile-first nature of modern web usage.

Test Frequency

Sites are retested periodically to track performance changes over time. High-ranking sites are tested more frequently.

Plugin Detection

Static Asset Analysis

We analyze publicly accessible CSS and JavaScript files to identify plugin signatures:

  • Plugin-specific asset paths (e.g., /wp-content/plugins/plugin-name/)
  • Unique CSS classes and JavaScript variables
  • Plugin-generated HTML comments and meta tags
  • WordPress action hooks and filter signatures

Detection Limitations

Our plugin count is an estimate because:

  • Some plugins don't leave public traces
  • Admin-only plugins may not be detected
  • Custom plugins without standard patterns are missed
  • Plugin asset optimization/minification can obscure signatures

Accuracy

Our detection typically captures 70-90% of active plugins that affect the frontend. The count represents plugins that have a measurable impact on page loading and rendering.

Ranking Algorithm

Efficiency Score Calculation

efficiency_score = psi_score / (1 + plugin_count * 0.1)

This formula rewards high PageSpeed scores while accounting for plugin complexity. Sites with fewer plugins need higher PSI scores to achieve the same efficiency rating.

Score Components

  • PSI Score (0-100): Direct performance measurement
  • Plugin Count: Estimated number of active plugins
  • Penalty Factor (0.1): How much each plugin reduces efficiency

Ranking Examples

High Efficiency

PSI: 95, Plugins: 5

Score: 95 / (1 + 5×0.1) = 63.3

Lower Efficiency

PSI: 85, Plugins: 20

Score: 85 / (1 + 20×0.1) = 28.3

Data Quality & Limitations

What We Measure

  • Real user performance metrics from Google's field data
  • Frontend-visible plugin implementations
  • Public theme information when available
  • Site accessibility and WordPress detection

Limitations

  • Performance can vary by geographic location and device
  • Plugin detection may miss backend-only or heavily cached plugins
  • Sites with aggressive caching may show inflated performance scores
  • CDN usage and hosting quality significantly impact results
  • Single-page applications may not represent typical WordPress patterns

Best Practices for Interpretation

  • Use rankings to identify patterns rather than absolute performance
  • Consider site type and complexity when comparing rankings
  • Look at trends over time rather than single measurements
  • Combine WP-Rank data with your own performance testing

API & Data Access

Public API Endpoints

GET /api/leaderboard - Paginated rankings
GET /api/sites/{domain} - Individual site details
GET /api/stats - Global statistics
POST /api/sites/submit - Submit site for analysis

Rate Limits

API requests are rate-limited to prevent abuse. Contact us if you need higher limits for research or integration purposes.

Data Updates

Rankings are recalculated daily. Site performance data is updated based on submission frequency and current ranking position.
